# Contributing to MarkOS MarkPanel

First off, thank you for considering contributing to MarkOS! It's people like you that make MarkOS such a great tool for the NAS community.

## How Can I Contribute?

### Reporting Bugs
* **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/mktt-ai-global/MarkOS-MarkPanel/issues).
* If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/mktt-ai-global/MarkOS-MarkPanel/issues/new).

### Suggesting Enhancements
* **Check if it has already been suggested** in the [Issues](https://github.com/mktt-ai-global/MarkOS-MarkPanel/issues).
* Explain why this enhancement would be useful to most MarkOS users.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Style Guide
* **Backend**: Follow PEP 8 for Python (FastAPI).
* **Frontend**: Use TypeScript, Tailwind CSS, and follow the [Next.js best practices](https://nextjs.org/docs/getting-started/project-structure).
* **Design**: Stick to the "Frosted Glass" (磨砂玻璃) UI tokens defined in `tailwind.config.ts`.
