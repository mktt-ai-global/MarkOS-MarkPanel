<h2 id="english">English</h2>

### Supported Versions

Only the latest version of MarkOS MarkPanel is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < 1.0   | :x:                |

### Reporting a Vulnerability

We take the security of MarkOS MarkPanel seriously. If you believe you have found a security vulnerability, please report it to us by:

1.  Opening a **Private Issue** on GitHub (if supported) or
2.  Emailing us at `security@markos.local` (Replace with your actual security contact email).

Please do **not** report security vulnerabilities via public GitHub issues.

#### What to include in your report

To help us triage and respond to your report, please include:

- A clear description of the vulnerability.
- Steps to reproduce (a proof-of-concept script or screenshot is highly appreciated).
- Potential impact if exploited.

<h2 id="中文">中文</h2>

### 受支持的版本

仅 MarkOS MarkPanel 的最新版本受安全更新支持。

| 版本 | 是否受支持 |
| --- | --- |
| v1.0.x | :white_check_mark: |
| < 1.0 | :x: |

### 报告漏洞

我们非常重视 MarkOS MarkPanel 的安全性。如果您认为发现了安全漏洞，请通过以下方式向我们报告：

1. 在 GitHub 上开启 **私有 Issue** (如果支持) 或
2. 发送邮件至 `security@markos.local` (请替换为您的实际安全联系邮箱)。

请**不要**通过公开的 GitHub Issue 报告安全漏洞。

#### 报告中应包含的内容

为了帮助我们分类和响应您的报告，请包含：

- 对漏洞的清晰描述。
- 重现步骤（非常欢迎提供概念验证脚本或截图）。
- 被利用后的潜在影响。
